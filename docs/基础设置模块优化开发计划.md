# 基础设置模块优化开发计划

更新时间：2026-06-21

## 一、本轮结论

本轮只做“基础设置模块优化”的开发计划，不在当前对话框编写代码。

基础设置定位为：

```text
主数据中心
口径管理中心
数据治理入口
```

核心原则：

1. 基础设置仍然只是一个一级导航。
2. 基础设置内部用 tabs 组织。
3. 其他模块如经营中心、数据中心、财务中心，本轮最多只允许调整筛选栏。
4. 禁止修改其他模块下方表格的固定结构。
5. 禁止修改经营汇总表计算、归集、固定行列和 HTML/CSS 主结构。

## 二、本轮允许范围

允许做：

1. 优化基础设置模块页面结构。
2. 整理基础设置内部 tabs。
3. 优化基础设置首页的数据体检展示。
4. 归拢公司架构、公司档案、名称口径、归集规则入口。
5. 其他模块只允许调整筛选栏。
6. 其他模块筛选栏可以改字段来源、字段顺序、字段样式。
7. 其他模块筛选栏可以调用 `filter_options_service.py`。
8. 其他模块筛选栏可以统一使用基础设置口径。

## 三、本轮禁止范围

禁止做：

1. 禁止修改其他模块下方表格的固定行标题。
2. 禁止修改其他模块下方表格的固定列标题。
3. 禁止修改其他模块下方表格的 HTML / CSS 主体结构。
4. 禁止修改经营汇总表固定行名称。
5. 禁止修改经营汇总表固定列名称。
6. 禁止修改经营汇总表计算公式。
7. 禁止修改经营汇总表归集规则。
8. 禁止修改 `src/operating_summary.py`。
9. 禁止修改数据导入解析核心逻辑。
10. 禁止修改财务报表计算逻辑。
11. 禁止顺手重构无关页面。
12. 禁止把基础设置拆成多个一级导航。

## 四、本轮基础设置页面目标结构

基础设置内部 tabs：

```text
首页
组织架构
公司档案
名称口径
归集规则
导入问题池
变更记录
```

## 五、首页：基础数据体检

首页展示基础数据体检。

至少展示：

| 指标 | 来源 | 说明 |
| --- | --- | --- |
| 公司数量 | `companies` | 当前启用或总公司数 |
| 公司别名数量 | `company_aliases` | 启用别名数 |
| 公司档案数量 | `dim_company` | 公司属性档案数 |
| 股权关系数量 | `ownership` | 股权/控制关系数 |
| 未分组公司数量 | `dim_company.business_group` | 空值或未分组 |
| 别名冲突数量 | `company_aliases` | 同一启用 alias 指向多个编码，理论应为 0 |
| 未识别公司名数量 | 后续问题池 | 当前无表时显示“待接入” |
| 公司档案缺失数量 | `companies` 对比 `dim_company` | 公司存在但档案缺失 |
| `tree_path` 异常数量 | `companies.tree_path` | 空路径、重复路径或父子异常 |

如果暂时没有底层数据：

```text
待接入
0
暂无数据
```

禁止编造数据。

## 六、组织架构 tab

负责：

```text
公司树
公司清单
公司清单导入
上下级关系
合并范围
股权/控制关系
```

复用：

```text
companies
ownership
src/company_hierarchy.py
src/company_structure.py
```

本轮只归拢入口和展示结构，不改底层公司树逻辑。

## 七、公司档案 tab

负责：

```text
业务模块
业态类型
区域
是否运营主体
未分组公司
```

复用：

```text
dim_company
src/company_dimension.py
```

页面必须提示：

```text
组织架构用于维护公司上下级和合并范围；
公司档案中的业务模块用于经营分析口径；
调整业务模块不要修改公司上级。
```

硬性要求：

1. 公司档案保存不得修改 `companies.parent_code`。
2. 公司档案保存不得重建公司树。
3. 公司档案只维护 `dim_company` 业务属性。

## 八、名称口径 tab

负责：

```text
公司别名
文件名识别
未识别公司名
冲突别名
别名导入/导出
```

复用：

```text
company_aliases
src/company_aliases.py
```

规则：

1. 模糊匹配只能作为建议。
2. 正式写入 `company_aliases` 必须由用户确认。
3. 同一个启用 `alias` 只能指向一个 `company_code`。
4. 停用别名应使用 `status = 0`，不要物理删除。

可以展示：


本轮禁止：

1. 禁止替换经营汇总表计算逻辑。
2. 禁止修改经营汇总表归集规则。
3. 禁止修改 `src/operating_summary.py`。
4. 禁止修改 `src/operating_reconciliation.py` 的计算规则，除非另起专项任务。

## 十二、其他模块筛选栏统一规则

如果本轮涉及经营中心、数据中心、财务中心，只允许调整筛选栏。

统一字段：

```text
period
year
month
company_code
include_descendants
business_group
business_type
region
data_status
```

字段来源：

| 字段 | 来源 |
| --- | --- |
| `period` | 事实数据已有期间 |
| `year` / `month` | 由 `period` 拆分或生成 |
| `company_code` | `companies` |
| `business_group` | `dim_company.business_group` |
| `business_type` | `dim_company.business_type` |
| `region` | `dim_company.region` |
| `data_status` | 报表或导入数据状态 |

筛选逻辑：

1. `period` 是最终查询字段。
2. `year` 和 `month` 只是 `period` 快捷筛选。
3. 经营单位选择母公司时，需要包含子孙公司。
4. 经营单位和所属板块同时选择时，必须取交集。
5. 筛选逻辑只能影响数据范围，不能影响表格固定行和固定列。


7. 其他模块下方表格固定行标题未改变。
8. 其他模块下方表格固定列标题未改变。
9. 经营汇总表仍然 8 列结构稳定。
10. 全量测试通过。
