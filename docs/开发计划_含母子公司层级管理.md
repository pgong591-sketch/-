# 财务报表数据仓库与报表生成系统 开发计划（含母子公司层级管理）

> **目标**  
> 构建一个轻量级财务数据库，集中存储 **40+ 家存在母子公司关系的企业** 每月多张标准报表数据，并在此基础上灵活组合生成各单体报表及层级汇总/合并报表。  
> **开发环境**：VSCode + Python 3.10+  
> **核心栈**：SQLite（前期无缝升级至 PostgreSQL） + Pandas + Streamlit（可选）

---

## 一、核心需求

### 1.1 数据源
- 每月收集各公司统一格式的 Excel 报表（共 10 类）：
  1. 资产负债表  
  2. 损益表  
  3. 现金流量表  
  4. 科目余额表  
  5. 收入成本费用明细表  
  6. 收入人次表  
  7. 非学科费用分配表  
  8. 管理中心部门收入成本费用表  
  9. 非学科管理中心部门收入成本费用表  
  10. 非学科课酬  

### 1.2 母子公司层级要求
- 公司之间为 **母子公司关系**，存在多层级控股。  
- 数据库需记录 **股权结构** 与 **组织层级**（集团 → 子集团 → 单体公司）。  
- 报表生成时需支持：
  - 按任意层级节点汇总（如某子公司及其下属所有公司合计）。  
  - 快速获取合并路径（从最底层向上逐级汇总）。  
  - 预留内部交易标识，为后续自动抵消打基础。

### 1.3 期望交付物
- 带层级关系的财务数据库（SQL 脚本 + Python 管理模块）  
- 自动化数据导入与校验工具  
- 报表查询引擎（可输出单体或层级汇总报表）  
- 可选：Streamlit 看板用于上传、查询、导出

## 二、数据库设计（核心：层级 + 财务数据）

### 2.1 设计思路
- **科目余额表** 为事实核心，多数标准报表可由其生成。  
- **专用明细表** 单独存放无法从科目余额派生的数据。  
- **公司表** 引入树形结构字段，支持任意层级汇总。  
- **股权关系表** 记录持股比例、控制权等信息，用于合并比例计算。  
- 所有财务数据表强制包含 `company_code`、`period`，建立复合索引。

### 2.2 核心表清单

#### ① 公司及层级

```sql
-- 公司基础信息表（支持树形层级）
companies (
    company_code   TEXT PRIMARY KEY,
    company_name   TEXT NOT NULL,
    parent_code    TEXT,              -- 母公司代码，顶级为 NULL
    level          INTEGER,          -- 层级深度（集团=0，一级子=1 ...）
    tree_path      TEXT,             -- 物化路径，如 '/ROOT/A/B/C' 方便查询子树
    is_leaf        BOOLEAN,          -- 是否叶子节点
    is_consolidated BOOLEAN DEFAULT 1, -- 是否纳入合并范围
    currency       TEXT DEFAULT 'CNY',
    industry       TEXT,
    created_date   DATE
);
```

```sql
-- 股权关系表（支持分段持股、多重持股）
ownership (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    parent_code     TEXT NOT NULL,              -- 母公司
    sub_code        TEXT NOT NULL,              -- 子公司
    ownership_pct   REAL NOT NULL,             -- 直接持股比例（%）
    effective_date  DATE NOT NULL,             -- 生效日期
    expiration_date DATE,                      -- 失效日期，NULL表示至今
    is_control      BOOLEAN DEFAULT 1,         -- 是否形成控制
    UNIQUE(parent_code, sub_code, effective_date)
);
```

#### ② 核心财务事实表

```sql
-- 科目余额表（主事实表，其他报表的基石）
account_balance (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    company_code    TEXT NOT NULL,
    period          CHAR(6) NOT NULL,          -- YYYYMM
    account_code    TEXT NOT NULL,
    account_name    TEXT,
    opening_balance DECIMAL(18,2),
    debit           DECIMAL(18,2),
    credit          DECIMAL(18,2),
    ending_balance  DECIMAL(18,2),
    assist_dims     TEXT,                      -- JSON格式辅助核算（客商、部门等）
    is_internal     BOOLEAN DEFAULT 0,         -- 是否关联交易科目，为抵消预留
    counterparty    TEXT,                      -- 内部交易对方公司代码
    UNIQUE(company_code, period, account_code)
);
```

```sql
-- 其他专用明细表（结构与原计划基本一致，增加公司、期间字段）
-- 以损益明细、收入人次为例：
pl_detail (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    company_code    TEXT NOT NULL,
    period          CHAR(6) NOT NULL,
    item_code       TEXT NOT NULL,
    item_name       TEXT,
    amount          DECIMAL(18,2),
    category        TEXT,                      -- 收入/成本/费用
    UNIQUE(company_code, period, item_code)
);

revenue_volume (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    company_code    TEXT NOT NULL,
    period          CHAR(6) NOT NULL,
    product_line    TEXT,
    customer_count  INTEGER,
    revenue_amount  DECIMAL(18,2)
);
```

*非学科费用分配表、管理中心部门收入成本费用表、非学科课酬表等均按此模式，所有表均包含 `company_code` + `period` 唯一约束。*

#### ③ 报表模板表（驱动资产负债表、损益表、现金流量表生成）

```sql
report_template (
    report_type     TEXT,                      -- 'BS', 'PL', 'CF'
    line_no         INTEGER,
    item_name       TEXT,
    debit_condition TEXT,                      -- 取数规则，如 'account_code BETWEEN 1001 AND 1999'
    credit_condition TEXT,
    sign            INTEGER DEFAULT 1,         -- 正负号
    is_summary_line BOOLEAN DEFAULT 0,         -- 是否小计行
    parent_line_no  INTEGER                    -- 上级小计行次
);
```

### 2.3 层级汇总设计
- **物化路径 `tree_path`**：在导入公司数据时自动维护，插入或变更父节点时更新整条子树的路径。查询某公司及其所有子孙公司时，使用 `tree_path LIKE '/ROOT/A/%'`，高效实现。
- **汇总函数**：Python 端编写工具函数，根据 `tree_path` 取出所有子节点公司代码，再汇总对应期间的财务数据。
- **股权比例权重**：如需合并抵消，可依据 `ownership` 表计算有效持股比例，但现在先聚焦数据层。

## 三、数据导入模块

### 3.1 解析器设计
针对 10 类报表，每类一个解析器类，均继承基类 `BaseParser`：
- `parse(filepath) -> pandas.DataFrame`  
- `validate(df) -> list of errors`  
- `save_to_db(df, period, company_code)`

**示例：科目余额表解析器规则**
- 识别 Excel 特定 sheet 或表头（如"科目编码""科目名称""期初余额"等）。
- 逐行读取，构造 `account_balance` 记录列表。
- 校验：`opening + debit - credit == ending`（允许小尾差），不平整条拒绝。
- 写入前检查唯一性，重复则更新（或跳过并报警）。

### 3.2 公司层级初始化
- 通过 `config/companies.xlsx` 维护公司信息与树形关系。  
- 在数据库中运行存储过程或 Python 脚本，自动计算 `level` 和 `tree_path`。  
- 股权关系表从另一配置 `config/ownership.xlsx` 导入。

### 3.3 批量导入流程
1. 用户将某月所有公司报表文件放入指定目录，文件名规范：  
   `{公司代码}_{期间YYYYMM}_{报表类型}.xlsx`  
2. Python 脚本遍历文件，通过文件名解析出公司、期间、报表类型，调用对应解析器。  
3. 所有文件处理完毕后，生成导入汇总报告（成功/失败/警告数量）。

## 四、报表生成与层级查询

### 4.1 单体报表
- 根据 `report_template` 定义的规则，从 `account_balance` 取数计算资产负债表、利润表、现金流量表。  
- 专用明细表（如收入人次）直接查询对应表。

### 4.2 层级汇总报表
- **指定某母公司及其下属所有子公司**：
  ```sql
  SELECT * FROM companies WHERE tree_path LIKE '/ROOT/A/%' OR company_code = 'A';
  ```
  得到公司列表，再对 `account_balance` 相同科目进行 `SUM` 汇总。  
- **多级汇总**：可设计函数 `get_consolidated_bs(company_code, period)`，自动计算该节点下所有子树的合并数（目前仅简单相加，后续可引入抵消）。  
- 输出选项：可导出为带层级的 Excel，如 Sheet1 为汇总结果，Sheet2 含各公司明细。

### 4.3 合并抵消预留
- 在科目余额表中已预留 `is_internal` 和 `counterparty` 字段。  
- 未来可编写模块，在层级汇总时自动提取内部交易记录，生成抵消分录并叠加到汇总数上。  
- 当前版本可先提供"汇总数"和"内部交易一览表"供人工核对。

## 七、注意事项

- **层级数据一致性**：修改公司父子关系或股权比例时，务必重算所有受影响子树的 `tree_path` 和 `level`。  
- **科目编码统一**：若各公司科目体系不同，需在导入时通过映射表统一，否则无法直接层级汇总。  
- **内部交易标记**：在导入科目余额表时，鼓励各公司填写内部客商，为日后的合并抵消打好基础。  
- **性能**：40 家公司 × 12 个月 × 约 500 科目 ≈ 24 万行科目余额，SQLite 足以应对。使用 `tree_path LIKE` 可高效进行子树查询。
