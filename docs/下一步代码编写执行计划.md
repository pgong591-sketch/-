# 下一步代码编写执行计划

更新时间：2026-06-12

## 一、本轮开发目标

本轮先做“月度财务数据收集闭环 MVP”，目标不是新增很多分析页面，而是让系统先能回答三个基础问题：

1. 本月哪些公司应该交哪些报表。
2. 哪些公司、哪些报表已经入库。
3. 哪些公司、哪些报表缺失、重复、异常，需要补交或重导。

同时把“新项目盈亏平衡测算”作为第二个并行模块做结构化落库准备，现有页面原型后续接数据库保存。

### 2. 建立月度收集规则表

优先在 `db/init.sql` 增加两张表：

```sql
CREATE TABLE IF NOT EXISTS monthly_collection_requirements (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    period TEXT NOT NULL,
    company_code TEXT NOT NULL,
    report_type TEXT NOT NULL,
    required INTEGER DEFAULT 1,
    due_date TEXT,
    remark TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(period, company_code, report_type)
);

CREATE TABLE IF NOT EXISTS monthly_collection_status (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    period TEXT NOT NULL,
    company_code TEXT NOT NULL,
    report_type TEXT NOT NULL,
    status TEXT DEFAULT '缺失',
    latest_batch_no TEXT,
    latest_file_name TEXT,
    latest_import_time TEXT,
    total_success_batches INTEGER DEFAULT 0,
    total_error_batches INTEGER DEFAULT 0,
    issue_detail TEXT,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(period, company_code, report_type)
);
```

含义：

- `monthly_collection_requirements`：定义“某期间、某公司、某报表类型是否应收”。
- `monthly_collection_status`：根据 `import_logs` 汇总出来的结果，可以重算。

    """根据 import_logs 重算某期间收集状态。"""

    """返回公司 x 报表类型的收集状态矩阵。"""

def get_collection_missing(period: str) -> pandas.DataFrame:
    """返回缺失、失败、重复导入清单。"""

状态规则先做简单版：

- 没有成功导入：`缺失`
- 有 1 条成功导入：`已收集`
- 有 2 条及以上成功导入：`重复`
- 有失败记录但无成功记录：`异常`

### 4. 在数据采集助手里加“月度收集总览”

修改文件：`app.py`

位置：`render_import()`。

把现有页面拆成两个 tab：

```python
tab_upload, tab_status = st.tabs(["文件采集入库", "月度收集总览"])
```

`文件采集入库` 保留现有上传、预检、入库、导入历史。

`月度收集总览` 新增：

- 期间输入框，默认取当前最新期间或手动输入。
- 报表类型多选，默认：科目余额表、资产负债表、损益表。
- “生成本月应收清单”按钮。
- “刷新收集状态”按钮。
- 状态矩阵表。
- 缺失/重复/异常明细表。

第一版只读展示即可，不急着做复杂编辑。

### 5. 增加测试

新增文件：`tests/test_monthly_collection.py`

测试重点：

1. 能创建月度收集表。
2. 能按公司和报表类型生成应收清单。
3. `import_logs` 没有记录时状态为缺失。
4. `import_logs` 有一条成功记录时状态为已收集。
5. `import_logs` 有多条成功记录时状态为重复。

测试不需要依赖真实生产库，使用临时 SQLite 或测试库。

## 四、本轮验收标准

本轮完成后，应能在页面中做到：

1. 输入期间，例如 `202603`。
2. 选择需要收集的报表类型。
3. 一键生成本期公司应收清单。
4. 一键刷新状态。
5. 看到每家公司每类报表的状态。
6. 单独看到缺失、重复、异常列表。

如果这些能跑通，第一阶段就有了“月度收集闭环”的骨架，后续再往里面接标准科目映射、模板自动填报、收入明细联动和盈亏平衡测算。
