# 第二阶段代码编写执行计划

更新时间：2026-06-12

## 一、启动条件

本阶段在“月度财务数据收集闭环 MVP”完成后启动。

启动前需要确认第一阶段已经做到：

1. 能生成某期间、某公司、某报表类型的应收清单。
2. 能根据 `import_logs` 判断已收集、缺失、重复、异常。
3. 数据采集助手页面能展示月度收集总览。
4. 新增测试通过，或至少新增模块测试通过。

## 二、本阶段目标

第二阶段目标是把已经收集进来的财务数据变成“可分类、可匹配、可填表”的数据。

核心目标：

1. 建立标准科目体系。
2. 建立本地科目到标准科目的映射关系。
3. 对科目余额表做自动归类。
4. 支持未映射科目检查。
5. 为后续特殊报表模板自动填入数据做准备。

这一阶段完成后，系统要能回答：

- 哪些科目已经映射到标准科目？
- 哪些科目还没有映射？
- 某个标准科目下面包含哪些原始科目？
- 某张特殊报表的某个单元格应该从哪些科目取数？

## 三、执行顺序

### 1. 检查现有表结构

先检查：

```powershell
rg -n "standard_accounts|account_mapping|template|科目" db src app.py tests
```

重点确认：

- `standard_accounts` 是否已有结构但数据为空。
- `account_mapping` 是否已有结构但数据为空。
- `balance_sheet_template`、`income_statement_template`、`cashflow_template` 的取数逻辑。
- `src/reports.py` 是否已经使用标准科目参与报表计算。

优先新增或完善模块：

```text
src/account_standardization.py
```

### 3. 在系统管理或数据采集助手中增加“科目映射维护”

修改 `app.py`。

建议先放在“系统管理”页面新增 tab：

```python
tab_account = st.tabs(["数据库状态", "公司管理", "科目映射", "关于"])
```

页面功能先做 MVP：

- 标准科目表展示。
- 科目映射表展示。
- 未映射科目清单。
- 映射关系编辑表。
- 保存映射按钮。

第一版不需要做复杂权限、版本审批。

### 4. 做未映射检查和数据质量提醒

在“月度收集总览”或“系统管理”里增加一个数据质量区块：

指标：

- 本期原始科目数。
- 已映射科目数。
- 未映射科目数。
- 映射覆盖率。

建议规则：

- 覆盖率 100%：正常。
- 覆盖率 95%-100%：提示关注。
- 覆盖率低于 95%：标记为待处理。

### 5. 预留特殊报表模板取数规则

新增表可以先设计，不一定本阶段全部做完：

```sql
CREATE TABLE IF NOT EXISTS special_report_templates (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    template_name TEXT NOT NULL,
    report_name TEXT NOT NULL,
    version TEXT DEFAULT 'v1',
    status INTEGER DEFAULT 1,
    remark TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS special_report_template_rules (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    template_id INTEGER NOT NULL,
    sheet_name TEXT,
    target_cell TEXT,
    target_field TEXT,
    rule_type TEXT NOT NULL,
    standard_account_code TEXT,
    account_name_keyword TEXT,
    company_scope TEXT,
    period_offset INTEGER DEFAULT 0,
    formula_expression TEXT,
    sign TEXT DEFAULT '+',
    sort_order INTEGER DEFAULT 0,
    remark TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (template_id) REFERENCES special_report_templates(id)
);
```

本阶段只需把设计记录下来，第三阶段再做模板填报。

## 五、验收标准

本阶段完成后，应能做到：

1. 看到标准科目表。
2. 看到每家公司本地科目到标准科目的映射关系。
3. 输入期间后，能列出还没有映射的科目。
4. 能在页面保存映射关系。
5. 能看到映射覆盖率。
6. 后续特殊报表自动填入有明确的数据基础。
