# UI 风格规范

更新时间：2026-06-27

## 总方向

整个系统 UI 风格向 iOS 端靠拢。

## 设计原则

1. 图标：优先使用简洁、线性、轻量的图标风格，避免厚重、杂乱、强装饰图标。
2. 按钮：按钮应克制、清爽、圆角适中，避免大面积高饱和亮蓝色块。
3. 文字：字号层级清楚，字重克制，整体接近 iOS 的清晰、轻量、现代感。
4. 间距：保持舒适留白，但不做松散的大块空白。
5. 色彩：以浅灰、深蓝、系统蓝为主，强调色少量使用。
6. 层级：一级操作、二级操作、普通信息要有明确视觉差异。

## 适用范围

- 左侧导航栏
- 顶部筛选区
- 表格工具栏
- 按钮和图标
- 报表卡片
- 文字标题与说明

后续新增页面或重构页面时，默认遵循本规范。

## 2026-06-29 执行记录

本轮先不改左侧导航栏，仅统一主界面的图标、按钮和文字风格：

1. 按钮改为更接近 iOS 的轻圆角、浅边框、低阴影样式。
2. 图标改为更轻量的线性观感，降低厚重感。
3. 页面标题、表单标签、指标卡文字调整为更清晰克制的字重和字号。

## 2026-07-01 同步记录

本轮同步经营汇总相关筛选和取数规则：

1. 筛选范围改为统一服务处理，支持集团、合并节点、公司、板块、业态、区域组合筛选。
2. 经营汇总表使用收入成本费用明细中的本月数和本年累计数，年度占比按累计金额计算。
3. 合并节点下对子公司“管理费服务费”和母公司“主营业务收入”做内部抵消，避免集团视角重复计算。
4. 页面继续沿用轻量 iOS 风格，补齐筛选标签、月份按钮、下载按钮和基础设置表格展示细节。

## 2026-07-02 全局 UI 调整记录

本轮按 iOS 风做全局视觉收敛：

1. 全局底色、边框、卡片阴影改为更轻的 iOS 风格。
2. 普通按钮、主按钮、下载按钮统一圆角、浅边框、低阴影和系统蓝。
3. 输入框、下拉框、文件上传区统一轻圆角和浅背景。
4. DataFrame、DataEditor、经营汇总 HTML 表格统一浅表头、弱边框和等宽数字。
5. Tab 改为轻量分段式视觉。
6. 左侧导航保持当前结构和宽度，只延续现有深蓝导航风格，不做交互改动。

## 2026-07-02 筛选区 iOS 风调整记录

本轮按最新样图统一全局筛选区：

1. 筛选卡片改为半透明白底、浅边框、轻阴影和 12px 圆角。
2. 输入框、下拉框统一 10px 圆角、浅白背景和更清晰的标签字重。
3. 查询按钮使用 iOS 系统蓝，重置和更多筛选使用白底浅边框。
4. 年份、月份、所属板块等选项改为轻量胶囊按钮，选中态为浅蓝底。
5. 更多筛选展开区自然换行铺开，减少小框横向拖动。
