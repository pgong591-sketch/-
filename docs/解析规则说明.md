# 报表解析规则说明

本文档描述每类报表解析器的输入 Excel 格式、输出字段和校验规则。

---

## 1. 科目余额表 (AccountBalanceParser)

| 项目 | 说明 |
|------|------|
| 表名 | `account_balance` |
| 文件名识别 | 含"科目"（如"科目余额表""科目辅助余额表"） |
| 表头行识别 | 第3行附近包含"科目编码""科目名称""期初余额"等关键字 |
| 公司期间提取 | 文件名前缀 `YYYYMM` + 中文公司名，或文件内指定 |
| 公司简称映射 | 从 `company_aliases` 表读取，配置化 |

### 输入列

| Excel 列名（可选变体） | 映射字段 | 说明 |
|---|---|---|
| 科目编码 / 科目代码 / 科目编号 | `account_code` | 文本 |
| 科目名称 / 会计科目名称 | `account_name` | 文本 |
| 期初余额 / 年初余额 | `opening_balance` | 数值 |
| 本期借方 / 借方金额 / 本月借方 | `debit_amount` | 数值 |
| 本期贷方 / 贷方金额 / 本月贷方 | `credit_amount` | 数值 |
| 期末余额 / 期末 | `ending_balance` | 数值 |
| 方向 / 借或贷 | `direction` | 文本 |
| 辅助核算 / 辅助项目 | `assist_dimensions` | 文本 |

### 输出字段

`company_code`, `period`, `account_code`, `account_name`, `opening_balance`, `debit_amount`, `credit_amount`, `ending_balance`, `direction`, `assist_dimensions`

### 校验规则

- 试算平衡：借方合计 = 贷方合计（绝对差值 `< 0.01`）
- 去重：同 `(company_code, period, account_code, assist_dimensions)` 唯一

## 2. 资产负债表 (BalanceSheetParser)

| 项目 | 说明 |
|------|------|
| 表名 | `balance_sheet` |
| 文件格式 | `.xls` 或 `.xlsx` |
| 文件名识别 | 含"资产负债表" |
| 模板格式 | 左右分栏：左侧资产 / 右侧负债和所有者权益 |

### 输入格式

固定 8 列：

| 列 | 内容 |
|----|------|
| A | 资产项目名称（如"货币资金""流动资产合计"） |
| B | 行次（左） |
| C | 期末余额（左） |
| D | 年初余额（左） |
| E | 负债和所有者权益项目名称 |
| F | 行次（右） |
| G | 期末余额（右） |
| H | 年初余额（右） |

### 输出字段

`company_code`, `period`, `side`（资产/负债和所有者权益）, `item_name`, `line_number`, `ending_balance`, `opening_balance`, `is_subtotal`（0=数据行/1=小计/2=section标题）, `sort_order`, `import_batch`

### 校验规则

- 资产总计 = 负债和所有者权益总计
- 同一行次在两侧分别唯一

## 3. 损益表 (IncomeStatementParser)

| 项目 | 说明 |
|------|------|
| 表名 | `income_statement` |
| 文件名识别 | 含"利润表"或"损益表" |
| 支持格式 | 单公司多期 / 多公司合集两种 |

### 格式 A：单公司多期

| 列 | 内容 |
|----|------|
| A | 项目名称（如"一、营业收入""减：营业成本"） |
| B..M | 各期金额（表头行为日期 YYYY-MM-DD） |
| 最后一列 | 本年累计（表头含"累计"） |

解析逻辑：
- 表头行日期匹配导入期间 → 取对应列作为 `period1_value`
- 最后一列（含"累计"）→ `cumulative_value`

### 格式 B：多公司合集

| 列 | 内容 |
|----|------|
| A | 项目名称 |
| B..N | 各公司列（表头为公司名），第一组 = 月度数，第二组同名列 = 累计数 |

解析逻辑：
- 公司名列去重：第一组 → `period1_value`，第二组同名 → `cumulative_value`
- 原始公司名保留在 `original_name` 字段

### 输出字段

`company_code`, `period`, `item_name`, `period1_value`, `cumulative_value`, `original_name`, `sort_order`, `import_batch`

### 校验规则

- 各期金额之和 = 本年累计（容差 `< 0.01`）

## 5. 收入人次表 (RevenueVolumeParser)

| 项目 | 说明 |
|------|------|
| 表名 | `revenue_volume` |
| 文件名识别 | 含"收入人次" |
| 样本文件 | `data/incoming/202603_revenue_volume.xls` |
| 状态 | 已验证样本结构；现有解析器和数据库表需扩展后再正式导入 |

### 样本工作表结构

样本包含 3 个工作表：

| 工作表 | 用途 | 处理口径 |
|---|---|---|
| `202603收入人次年级表` | 按校区、年级、月份透视汇总 | 只用于校验，不作为明细入库源 |
| `202603收入人次科目表` | 按校区、科目、月份透视汇总 | 只用于校验，不作为明细入库源 |
| `202603收入人次表` | 明细数据源 | 后续规律分析和入库应以此表为准 |

### 明细输入列

| Excel 列名 | 建议字段 | 说明 |
|---|---|---|
| 年份 | `year` | 业务年份 |
| 月份 | `month` | 业务月份；样本覆盖 1、2、3、4 月 |
| 季度 | `source_quarter_label` | 源系统季度/班期标签，如寒假班、春季班、秋季班、暑假班、待定；不是自然季度 |
| 校区名称 | `campus_name` | 原始校区名称，后续需映射到公司/校区维度 |
| 年级 | `grade` | 可为空；空值需单独保留，不要误填为某个年级 |
| 科目 | `subject` | 可为空；空值需单独保留，不要误填为某个科目 |
| 人次_次数 | `visit_count` | 人次/次数，整数 |
| 实际收入 | `revenue_amount` | 实际收入金额 |

### 标准化输出建议

后续用于规律分析时，不建议直接写入当前简化版 `revenue_volume` 表。建议先使用分析明细结构：

`data_period`, `business_period`, `year`, `month`, `calendar_quarter`, `source_quarter_label`, `campus_name`, `company_code`, `grade`, `subject`, `visit_count`, `revenue_amount`, `unit_price`, `source_file`, `source_sheet`

字段说明：

- `data_period`：来自文件名的批次/数据包期间，例如 `202603`。
- `business_period`：逐行按 `year + month` 生成，例如 `202601`、`202602`、`202603`、`202604`。
- `calendar_quarter`：自然季度标签，例如 1 至 3 月为 `2026Q1`，4 月为 `2026Q2`。
- `source_quarter_label`：源表原始“季度”列，保留寒假班、春季班、秋季班、暑假班、待定等班期标签。
- `unit_price`：`revenue_amount / visit_count`，仅作为派生分析字段。

### 样本验证结果

- 明细表共 4203 行，总收入 `96029045.37`，总人次 `661351`。
- 覆盖 38 个校区、20 个非空年级、39 个非空科目。
- 源季度/班期标签分布：寒假班、春季班、秋季班、暑假班、待定。
- 按明细表重新汇总后，可完全还原“收入人次年级表”和“收入人次科目表”，校区+年级、校区+科目两类透视校验差异均为 0。

### 当前结构缺口与注意事项

- 当前 `revenue_volume` 表只有 `company_code`、`period`、`product_line`、`customer_count`、`revenue_amount`、`unit_price`，无法承载校区、年级、科目、源季度标签和自然季度标签。
- 当前 `RevenueVolumeParser` 默认读取第一个工作表，会误读透视汇总页；正式导入时应优先识别并读取明细工作表 `*收入人次表`。
- 文件名期间 `202603` 不能简单等同于所有行的业务月份；样本实际包含 2026 年 1 至 4 月数据。
- 源表列名“季度”不是自然季度，应在系统中命名为 `source_quarter_label` 或 `term_label`，自然季度另行派生为 `calendar_quarter`。
- 样本存在空年级 261 行、空科目 51 行、收入为 0 但人次大于 0 的记录 8 行；这些应作为有效异常/待确认项保留，不应静默删除。
- 已按用户确认补充本样本校区映射：初中总部校区 → 莞城初中部（`101010128`），小学总部校区 → 莞城小学部（`101010120`），深圳福田校区 → 深圳卓越（`1010201`），茶山校区 → 茶山学前校区（`101010131`），西平小初校区 → 西平校区（`101010106`）。
- 班德校区按用户确认暂不处理，保留为未匹配；仍待确认的样本校区包括东莞校区、寮步校区、广东省社区少年宫莞城（多维宫）、松山湖旗舰中心、红树湾校区、轻子校区、高中总部校区等。

## 通用规则

### 文件名 → 公司/期间提取

- 优先格式：`YYYYMM中文公司名报表类型.xlsx` → 提取 YYYYMM 和公司名
- 备选格式：`中文公司名_YYYYMM.xlsx`
- 提取的公司名依次经：公司别名表 → 精确匹配 → 模糊匹配

### 导入重复检查

同 `(company_code, period, report_type)` 已存在成功记录则拒绝

### 公司简称映射

优先级：`company_aliases` 表（配置化）> 公司名称精确匹配 > 公司名称模糊匹配（子串包含）
