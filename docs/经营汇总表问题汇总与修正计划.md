# 经营汇总表问题汇总与修正计划

更新时间：2026-06-17

## 一、结论

最近一次经营汇总表页面效果差，不是单纯样式问题，而是页面入口、数据口径和开发计划执行顺序混在了一起。

下一轮先不继续加新功能，先把经营汇总表收成一个可用版本：

1. 只保留一个主入口：经营中心 -> 经营汇总表。
2. 只认一个主口径：收入成本费用表分类汇总公式结果 -> 经营汇总表。
3. 取消宽表方向，主页面改为“经营摘要表 + 主体拆分 + 来源下钻”。
4. 旧的损益表口径和 Excel 模板预览只作为核对入口，不再放在主流程里。

## 二、当前问题汇总

### 1. 页面入口重复，用户看到的“经营汇总表”不是同一张表

当前代码里至少有两套经营汇总相关页面：

- `render_profit_original_table()`：使用 `src/operating_summary.py`，原计划暂按 `pl_detail` 取数，但后续应改成收入成本费用表分类汇总结果。
- `render_multi_operating_summary()`：使用 `src/multidim_reports.py`，来源是 `income_statement`，同时还塞了驾驶舱、费用分析和模板预览。

侧边栏里现在显示的是“利润表明细（原表）”，但标签叫“经营汇总表”。同时代码里还保留了“多维经营汇总表”的页面映射。

影响：

- 用户不知道自己点进去的是新经营汇总表，还是旧多维经营汇总表。
- 同名页面展示口径不同，金额可能对不上。
- 后续修 bug 容易修错入口。

### 2. 数据来源混用，经营口径不稳定

之前计划明确写过：

```text
收入成本费用表
-> 分类汇总公式区
-> 分类汇总结果
-> 经营汇总表指标
```

但当前实现里：

- 新经营汇总表从 `pl_detail` 取数，且当前 `pl_detail` 为空。
- 旧经营汇总/驾驶舱从 `income_statement` 取数。
- 费用分析页面仍按损益表项目拆：营业收入、营业成本、管理费用、财务费用等。

影响：

- “经营汇总表”的费用结构和公式文档不一致。
- 同一期间、同一公司范围，不同 tab 金额可能不一致。
- 页面看起来像经营汇总，但实际可能还是损益表汇总。
- 收入成本费用表里已经有分类汇总公式结果，但系统没有把这块作为主取数来源。

### 3. 本年累计没有真正实现

计划要求经营汇总表展示：

- 本月数
- 占费用比例
- 占收入比例
- 本年累计数
- 累计占费用比例
- 累计占收入比例

当前 `src/operating_summary.py` 里 `get_operating_summary_source_detail()` 把：

```text
ytd_amount = amount
```

也就是本年累计等于本月数。

影响：

- 累计列如果后续展示，会是错的。
- 当前导出的“2026合计”也容易误导用户。
- 经营分析不能用于正式复核。

### 4. 主页面缺少主体拆分，但不再走宽表方案

旧计划曾要求保留 Excel 宽表阅读习惯，但该方向取消。

当前新经营汇总表只展示总表：

```text
费用科目、合计、占费用比、占收入比、上月、环比、备注
```

虽然底层行数据里有 `companies`，但页面没有提供清晰的主体拆分视图。

影响：

- 用户只能看合计，看不到金额主要来自哪些主体。
- 如果不做主体拆分，经营异常很难定位。

### 5. 下钻入口是假入口

当前金额显示成可点击样式，但代码最后只清理 query 参数，没有真正弹出来源明细。

计划要求：

```text
点击“人工” -> 展示工资、奖金、社保、社保费、公积金、福利费、教师福利
点击“其他” -> 展示成本费用合计中未被明确归集的明细项目
```

影响：

- 页面暗示能下钻，但点了没有实质结果。
- 财务复核时无法追溯金额来源。

### 6. 旧模板预览仍在主流程里

之前“数字看不见”的计划针对的是：

```text
经营中心 -> 多维经营汇总表 -> 模板预览
```

这本质是 Excel 模板转 HTML 的可读性问题。

但新计划已经明确经营汇总表不应只展示原 Excel 模板，而应按归集规则生成。模板预览可以保留，但不能作为主经营汇总页面的一部分。

影响：

- 用户会继续被旧模板样式问题干扰。
- 修样式不能解决经营汇总表本身的口径问题。

### 7. 页面状态文案写死

当前页面显示：

```text
数据状态：已审核
更新时间：2026-03-31 23:59:59
```

这不是从导入日志或数据库状态计算出来的。

影响：

- 用户会误以为数据已审核。
- 实际导入失败、缺数据、未刷新时也可能显示正常。

## 三、和之前计划的对比

| 原计划 | 当前实现 | 修正判断 |
| --- | --- | --- |
| 主数据来源是收入成本费用表分类汇总公式结果 | 新旧页面分别使用 `pl_detail` 和 `income_statement` | 必须新增/改造分类汇总结果解析，不能用普通损益表顶替 |
| 经营汇总表不只展示 Excel 模板 | 旧页面仍把模板预览放在主流程 | 模板预览降级为核对入口 |
| 旧计划曾要求保留 Excel 宽表阅读习惯 | 用户已取消宽表方向 | 改为摘要表 + 主体拆分，不做主体横向宽表 |
| 支持本月和本年累计 | 当前累计等于本月 | 必须补真实累计逻辑 |
| 支持来源下钻 | 当前只是点击样式 | 必须做真实明细抽屉/弹窗 |
| 页面先不重做视觉，只先接规则和取数 | 实际混入驾驶舱、费用分析、模板预览 | 先收口，再美化 |

## 四、修正后的开发顺序

### 第 1 轮：收口入口和口径

目标：让用户只看到一张可信的经营汇总表。

改动：

1. 侧边栏只保留“经营汇总表”一个主入口。
2. 主入口绑定 `render_profit_original_table()`。
3. `render_multi_operating_summary()` 暂时从菜单移除，或改名为“旧版经营分析（待下线）”。
4. 利润驾驶舱、费用分析如果继续保留，必须标清“损益表口径”，不要叫经营汇总口径。
5. 经营汇总表取数入口改成收入成本费用表分类汇总结果；结果未入库时显示诊断，不用 `income_statement` 顶替。

验收：

- 用户点击经营中心里的“经营汇总表”，只进入收入成本费用表分类汇总口径页面。
- 页面上不再同时出现旧模板预览、旧损益汇总和新经营汇总混排。

### 第 2 轮：补真实累计和主体拆分

目标：让表能看清总数，也能看清金额来自哪些主体，但不做横向宽表。

改动：

1. 解析收入成本费用表分类汇总区里的本月数和本年累计数。
2. `build_operating_summary_rows()` 同时输出本月和本年累计。
3. 经营摘要表列调整为：

```text
费用科目
本月合计
本月占费用比
本月占收入比
本年累计
累计占费用比
累计占收入比
备注
```

4. 另做“主体拆分”区域：

```text
主体
收入合计
成本费用合计
净利润
净利率
主要费用项
```

5. 费用项点击后，再展示该费用项按主体拆分，不在主表里横向铺开。

验收：

- 本月数和累计数来自收入成本费用表分类汇总区，不再默认相等。
- 能看到主体拆分排名。
- 点某个费用项后，能看到该费用项按主体拆分。
- 导出 Excel 包含同页面一致的列。

### 第 3 轮：补来源下钻

目标：点金额能查来源。

改动：

1. 点击归集项目，展示来源明细：
   - 公司
   - 期间
   - 来源项目
   - 费用段
   - 本月金额
   - 本年累计金额
   - 导入批次
2. “其他”用剩余法明细解释：列出未被明确归集的项目。
3. 下钻先用页面内明细区，不做复杂跳转。

验收：

- 点“人工”能看到工资、奖金、社保、公积金等来源。
- 点“其他”能看到未归集项目和金额差额。

### 第 4 轮：再做视觉调整

目标：在口径正确后再把页面做顺。

改动：

1. 固定左侧费用科目列。
2. 固定表头。
3. 合计行、利润行、异常行突出。
4. 空数据时显示应导入什么表，不显示假审核状态。
5. 旧模板预览只保留在“模板核对”或“系统管理”里。

验收：

- 页面不再像多个方案拼在一起。
- 数字清楚、列不挤、空数据提示能指导下一步。

## 六、暂停项

以下内容暂时不要做：

1. 新增经营驾驶舱图表。
2. 继续优化旧 Excel 模板预览。
3. 新增特殊报表自动填表。
4. 扩展盈亏平衡测算。

原因：经营汇总表主流程未收口前，继续加功能只会让页面更乱。
