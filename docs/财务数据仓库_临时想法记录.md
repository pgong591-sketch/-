记录日期：2026-06-09

本文档用于保存关于财务数据仓库、特殊报表、经营分析和风险监控的临时想法。后续每次有新的想法，都追加到本文档，避免只保留在对话中导致丢失。

## 一、初期目标：每月财务报表数据归集与分类

建立一个财务数据仓库，能够收集、整理每月所有财务报表数据，并按报表类型、月份、主体、科目、业务类别等维度进行分类归档，为后续查询、分析、对比和自动生成报告打基础。

早期能力包括：

- 数据收集：支持导入每月的资产负债表、利润表、现金流量表、科目余额表、费用明细、收入成本等财务数据。
- 分类整理：按月份、公司或部门、报表类型、科目、项目、业务线等维度统一归类，避免每个月数据散落在不同 Excel 中。
- 分析基础：数据结构要支持后续同比、环比、预算执行、异常波动、自动月报等功能。

一句话目标：先把每月所有财务报表数据统一收进来、归好类、存清楚。

## 二、特殊数据报表：按模板自动匹配填入

在数据仓库完成每月财务数据归集和分类后，系统能够根据预先设定好的报表模板、科目映射关系和取数规则，自动匹配对应数据并填入指定表格，生成所需的特殊数据报表。

核心能力包括：

- 预设模板：提前准备目标报表格式，例如特殊管理报表、集团上报表、项目分析表、专项统计表等。
- 建立映射关系：明确每个表格单元格或字段应从哪个数据源、哪个月份、哪个主体、哪个科目、哪个分类中取数。
- 自动匹配填入：系统按规则自动查找数据、汇总计算、填入模板，减少手工复制粘贴。
- 生成特殊报表：最终输出符合固定格式要求的 Excel、网页表格或可下载报表。

一句话目标：不仅要把数据收好，还要能按预设模板自动取数填表，生成指定格式的特殊财务报表。

## 三、经营方案分析：收入数据、财务数据和业务方案联动

后续要把详细收入数据和财务数据链接起来观看，并根据某个时间给出的业务方案，推断该方案的回报率和效率值，判断该方案到底能不能达到提升销售额或提升利润的目的，以及是否形成正反馈。

同时，系统应能联动分析某段时间、某个期间内公司的经营情况。

核心能力包括：

- 收入与财务联动：把销售收入明细、客户、产品、项目、区域等业务数据，和利润、成本、费用、现金流等财务数据连接起来看。
- 方案效果评估：针对某个时间点实施的业务方案，分析实施前后销售额、毛利、净利、费用率、回款、库存、周转等指标变化。
- 回报率和效率判断：计算方案带来的收益和投入之间的关系，例如 ROI、利润提升率、费用效率、销售转化效率等。
- 正反馈判断：判断方案是提升了销售额、提升了利润，还是只增加收入但压低利润，或者短期有效但长期无效。
- 期间经营分析：能按某段时间联动分析公司的整体经营情况，看到收入、利润、成本、费用、现金流、业务动作之间的关系。
- 新项目盈亏平衡测算：针对新项目、新校区、新业务线或新投资方案，测算固定成本、变动成本、预期收入、毛利率、费用投入、回本周期和盈亏平衡点，判断项目需要达到多少收入、客户量或业务规模才能不亏损。

一句话目标：把业务动作和财务结果连起来，判断某个方案到底有没有真正改善公司的经营表现。

## 四、风险资产与风险投资监控

建立风险资产与风险投资监控模块，对公司重点风险项目、异常资产、投资事项和特殊关注项目进行持续跟踪，并结合财务知识、经营经验和商业历史案例，给出参考判断、风险提示和处理建议。

核心能力包括：

- 风险资产监控：跟踪应收账款、存货、长期挂账、异常预付款、低效资产、闲置资产、减值风险资产等。
- 风险投资跟踪：跟踪投资项目的投入金额、回报情况、现金流、进度、风险敞口、退出可能性和实际收益偏差。
- 特殊关注项目建议：对被标记为重点关注的项目，整理财务表现、经营风险、异常变化，并给出参考意见。
- 财务与经营经验辅助：判断时不仅看数字，还要结合常见财务风险、企业经营经验、资金周转逻辑、利润质量、现金流压力等。
- 商业历史案例借鉴：查找或整理商业历史上的类似事件，例如投资失败、库存积压、回款恶化、盲目扩张、利润虚高等案例，并说明后来结果，用来辅助判断当前项目。

一句话目标：不仅要看公司现在的数据，还要识别潜在风险，并借助财务经验和历史案例，判断哪些资产或投资可能出问题、该怎么提前处理。

## 2026-06-12 技术路线补充：数据库后台与界面设计顺序

- 数据库后台优先考虑 Supabase。后续如果要从本地 SQLite 走向云端、权限、多人协作、文件/认证/边缘函数等一体化后台，默认先按 Supabase 方案评估。
- 只有在需求明确是“纯 Postgres 托管数据库”、不需要 Supabase 后台能力时，再考虑 Neon 作为 Postgres 托管选项。
- 界面设计先在当前 Streamlit 应用里打磨真实业务窗口、筛选区、表格、KPI 卡片和操作流程；等本地界面形态稳定后，再进入 Figma 做系统化视觉稿和组件沉淀。
