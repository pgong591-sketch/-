# 数据库同步自动化方案

本项目采用“代码 Git 同步 + 数据库快照同步”的方式。

真实的 `data/finance_dw.db` 不进入 Git。办公室电脑和家里电脑通过共享目录交换压缩后的 SQLite 快照包。脚本不会直接在网盘目录里运行数据库，也不会尝试自动合并两份同时被修改过的数据库。

## 推荐共享目录

任选一种两台电脑都能访问的位置：

- OneDrive / 坚果云 / NAS 的一个专用文件夹
- 移动硬盘里的专用文件夹
- Tailscale 或局域网共享目录

建议目录名类似：

```text
D:\finance_dw_sync
```

共享目录里只放脚本生成的 `finance_dw_snapshot_*.zip`，不要把正在运行的 `finance_dw.db` 直接放进去。

## 首次配置

办公室电脑：

```powershell
python scripts\db_sync.py init --sync-dir "D:\finance_dw_sync" --machine office
```

家里电脑：

```powershell
python scripts\db_sync.py init --sync-dir "D:\finance_dw_sync" --machine home
```

`init` 会写入本机专用的 `.dbsync.local.json`，该文件已被 `.gitignore` 排除。

## 日常使用

离开办公室前，发布办公室当前数据库：

```powershell
python scripts\db_sync.py push --note "leaving office"
```

到家后，拉取最新数据库：

```powershell
git pull
python scripts\db_sync.py pull
```

离开家前，发布家里当前数据库：

```powershell
git push
python scripts\db_sync.py push --note "leaving home"
```

回到办公室后，拉取最新数据库：

```powershell
git pull
python scripts\db_sync.py pull
```

## Codex 口令

你可以直接说：

```text
我回家了，帮我同步数据库
```

Codex 应执行：

```powershell
git pull
python scripts\db_sync.py status
python scripts\db_sync.py pull
```

你也可以说：

```text
我要下班了，帮我发布数据库
```

Codex 应执行：

```powershell
python scripts\db_sync.py status
python scripts\db_sync.py push --note "leaving office"
```

## 安全机制

- `push` 使用 SQLite backup API 生成一致性快照。
- 快照包包含 `manifest.json`，记录来源机器、时间、哈希和表记录数。
- `pull` 会先校验快照哈希和 SQLite 完整性。
- `pull` 覆盖本机数据库前，会自动备份到 `data/backups/db_sync/`。
- 如果本机数据库存在未同步修改，`pull` 默认拒绝覆盖。确认要覆盖时再使用 `--force`。

## 冲突规则

这套方案适合“同一时间主要只在一台电脑上工作”的场景。

如果办公室和家里都基于同一个旧版本分别修改了数据库，脚本不会自动合并，因为 SQLite 文件级合并风险很高。此时应先保留两边备份，再人工决定以哪一份为准，或写专门的数据合并脚本。
