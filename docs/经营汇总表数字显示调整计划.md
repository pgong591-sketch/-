# 经营汇总表数字显示调整计划

更新时间：2026-06-12

## 一、问题描述

经营中心里的“经营汇总表”页面数字完全看不见，影响报表查看。

当前入口：

- 页面函数：`app.py` 的 `render_multi_operating_summary()`
- 渲染函数：`app.py` 的 `_render_fixed_template_sheet()`
- 底层转换：`src/template_workbook.py` 的 `load_template_sheet()` 和 `_sheet_to_html()`
- 页面调用：`_render_fixed_template_sheet("经营汇总表", "multi_operating_template")`

初步判断：经营汇总表是从 Excel 模板转换成 HTML 展示，数字不可见大概率不是数据缺失，而是样式问题。可能原因包括：

1. Excel 模板里的字体颜色被原样带入 HTML 后，与网页背景色对比度太低。
2. 某些数字单元格字体是白色或接近背景色。
3. 单元格背景色、主题色、字体色转换后出现异常。
4. 表格字体过小、列宽不足、溢出隐藏，导致数字看起来像没有显示。

## 二、调整目标

第一目标：让经营汇总表的所有数字清晰可见。

第二目标：修复方式尽量放在模板转 HTML 的公共层，避免只针对一个页面硬改。

第三目标：不破坏原 Excel 模板的大体版式，包括合并单元格、边框、背景色、加粗和对齐。

## 三、具体调整计划

### 1. 先确认是样式问题还是数据问题

在代码窗口先检查：

```powershell
rg -n "经营汇总表|multi_operating_template|_sheet_to_html|_cell_style" app.py src
```

然后在本地页面打开：

```powershell
streamlit run app.py
```

进入：

```text
经营中心 -> 多维经营汇总表
```

确认：

- 表格是否正常加载。
- 行列是否存在。
- 表头是否能看见。
- 数字区域是否只是颜色看不见。
- 鼠标选中数字区域后是否能看到文本。

如果鼠标选中后能看到数字，基本确认是字体颜色/背景色对比度问题。

### 2. 在模板 HTML 转换层增加可读性保护

主要修改文件：

```text
src/template_workbook.py
```

建议新增函数：

```python
def _relative_luminance(hex_color: str) -> float:
    """计算颜色亮度。"""

def _contrast_ratio(foreground: str, background: str) -> float:
    """计算前景色和背景色对比度。"""

def _readable_text_color(text_color: str | None, background_color: str | None, is_numeric: bool = False) -> str | None:
    """当字体色和背景色对比度过低时，返回更清晰的字体色。"""
```

规则建议：

- 如果字体色为空，保留默认文字色。
- 如果背景色为空，默认背景按 `#ffffff` 处理。
- 如果对比度低于 4.5，自动改成深色 `#111827` 或白色 `#ffffff`。
- 数字单元格优先使用深色字体，保证在浅色背景下清楚。
- 深色背景则使用白色字体。

### 3. 修改 `_cell_style()`

当前 `_cell_style()` 会直接把 Excel 字体颜色转成 CSS：

```python
color = _color_to_hex(font.color, theme_colors) if font.color else None
if color:
    style.append(f"color:{color}")
```

建议改成：

```python
background_color = ...
font_color = ...
safe_color = _readable_text_color(font_color, background_color, is_numeric=is_numeric)
if safe_color:
    style.append(f"color:{safe_color}")
```

其中 `is_numeric` 可以根据 `value_cell.value` 判断：

```python
is_numeric = isinstance(value_cell.value, (int, float)) and not isinstance(value_cell.value, bool)
```

### 4. 给模板表格增加基础可读性 CSS

继续修改 `src/template_workbook.py` 里的 `_sheet_to_html()`。

建议增强 `.template-sheet td`：

```css
.template-sheet td{
    box-sizing:border-box;
    border:1px solid transparent;
    padding:4px 8px;
    min-width:48px;
    line-height:1.35;
    white-space:pre-wrap;
    overflow:visible;
    color:#111827;
}
.template-sheet td.numeric-cell{
    font-variant-numeric:tabular-nums;
}
```

如果不想改 class 结构，也可以先只调整默认 `td` 的 `color`、`padding`、`min-width`、`line-height`。

### 5. 可选：给数字单元格加 class

在 `_sheet_to_html()` 生成单元格时，判断是否是数字：

```python
classes = []
if isinstance(value_cell.value, (int, float)) and not isinstance(value_cell.value, bool):
    classes.append("numeric-cell")
class_attr = f' class="{" ".join(classes)}"' if classes else ""
```

然后输出：

```python
cells.append(f'<td{class_attr}{span_attrs} style="{...}">{value}</td>')
```

这样后续可以单独优化数字展示。

### 6. 增加测试

新增或修改：

```text
tests/test_template_workbook.py
```

测试重点：

1. 字体色和背景色对比度过低时，会自动替换成可读颜色。
2. 浅色背景下白色字体会被修正。
3. 深色背景下深色字体会被修正。
4. 数字单元格不会因为 Excel 模板字体色异常而不可见。

如果现有测试不方便构造 workbook，可以先给颜色函数做单元测试。

## 五、验收标准

修复完成后应满足：

1. “经营中心 -> 多维经营汇总表”数字清楚可见。
2. 表格背景色和边框仍基本保持 Excel 模板风格。
3. 其他固定模板页面，例如“损益表”“图片简报”，不应出现明显样式退化。
4. 新增颜色可读性测试通过。

## 六、后续延伸

如果这个问题在多个固定模板页都出现，后续可以增加一个“模板显示模式”：

- 原始模板模式：尽量还原 Excel。
- 可读增强模式：优先保证网页阅读。
- 打印预览模式：适合导出或截图。
