## 一、最值得借鉴的页面结构

经贝管家的核心不是单个页面好看，而是把经营分析做成了一个固定工作台结构：

1. 左侧一级业务模块
   - 经营中心
   - 费控报销
   - 财务中心
   - 发票中心
   - 基础设置

2. 左侧二级业务菜单
   - 驾驶舱
   - 管理报表
   - 报表数据处理
   - 数据采集助手

3. 主页面顶部筛选区
   - 预算
   - 统计周期，支持开始月份到结束月份
   - 经营单元，多经营单元选择

4. 页面主体
   - 指标卡
   - 图表区
   - 表格区
   - 明细追溯区
   - 导出入口

我们系统应优先学习这个结构，而不是先堆功能。

## 三、筛选区借鉴方案

### 3.1 经贝筛选区特点

经贝的筛选区有三个优点：

1. 固定在页面顶部，用户先选范围再看结果。
2. 筛选项不是很多，默认只展示最关键字段。
3. 业务语言清楚，例如预算、统计周期、经营单元。

### 3.2 我们系统建议标准筛选条

驾驶舱和管理报表建议统一使用以下筛选条：

| 筛选项 | 控件 | 说明 |
| --- | --- | --- |
| 预算方案 | 下拉框 | 可为空，第一阶段可先隐藏或显示未设置 |
| 统计周期 | 起止月份 | 支持单月和多月区间 |
| 经营单元 | 多选 | 公司、校区、板块或集团范围 |
| 经营口径 | 分段按钮 | 默认公司、集团汇总、板块汇总 |
| 更多筛选 | 折叠区 | 年份、月份、所属板块、业态、区域 |
| 查询 | 主按钮 | 刷新当前页面数据 |
| 重置 | 次按钮 | 恢复默认条件 |
| 导出 | 次按钮 | 放在报表页右上角 |

### 3.3 筛选交互规则

- 首页默认只显示一行筛选，不要一上来铺满条件。
- “更多筛选”展开后显示年份、月份、所属板块等快速筛选。
- 经营单元要支持多选和搜索，后续公司数量多时很关键。
- 查询条件应在各页面保持一致，避免每个页面重新选。
- 筛选区不要和图表卡片混在一起，应作为独立工具条。

## 四、管理报表借鉴方案

### 4.1 贡献式利润表

经贝的常规贡献式利润表有几个值得借鉴的点：

- 同一页面内用页签切换视角。
- 有单元视角和月度视角。
- 月度视角按月份横向展开。
- 指标包含实绩、占总收入比、去年实绩、去年占比。
- 可展开收入、成本费用、经营利润、经营净利润等层级。

我们第一阶段建议做：

1. 常规贡献式利润表
   - 单元维度：按公司/校区/板块展示。
   - 月度维度：按月份横向展示。
   - 指标：收入、成本费用、经营利润、经营净利润、收入占比。

2. 预实对比
   - 预算数
   - 实际数
   - 差异额
   - 完成率
   - 同比数据可放第二阶段。

### 4.2 产品贡献式利润表

经贝的产品贡献式利润表把列维度从经营单元换成产品线，筛选区增加业务科目。

我们可以借鉴为：

- 产品线贡献表
- 课程/项目贡献表
- 科目/业务线贡献表

第一阶段不需要做很复杂，先支持：

- 产品线
- 收入
- 成本费用
- 经营利润
- 收入占比
- 利润率

前提是后续需要补齐收入明细、产品线、业务科目等维度数据。

### 4.3 资金变动表

经贝资金变动表有两个视角：

- 月度
- 维度

筛选项从经营单元变为账户名称，表格展示期初余额、余额等项目。

我们可以把它和已有的资金预警表结合：

- 资金变动表
- 资金预警表
- 可使用周转金
- 账户余额趋势
- 往来款占用

这块建议排在贡献式利润表之后，因为它需要更清楚的资金账户和往来口径。

## 五、报表数据处理借鉴方案

### 5.1 核算记录

经贝的核算记录页是“数据追溯中心”：

- 顶部有总收入、总支出、余额、总往来。
- 表格记录每条核算流水。
- 字段包含经营单元、流水单号、分摊状态、数据来源、业务科目、金额、产品线、项目、记账日期、核算方式等。

我们系统应建立自己的“核算记录”页面：

| 字段 | 来源 |
| --- | --- |
| 期间 | 导入文件或系统生成 |
| 公司/经营单元 | 公司主数据 |
| 数据来源 | 科目余额表、损益明细、收入人次、手工补录 |
| 报表类型 | 导入类型 |
| 科目编码/科目名称 | 财务科目 |
| 标准科目 | 科目映射 |
| 业务科目 | 管理口径 |
| 金额 | 事实数据 |
| 产品线/项目 | 业务维度 |
| 导入批次 | import_logs |
| 核算方式 | 自动、手工、调整 |

第一阶段先做只读追溯，不做复杂编辑。

### 5.2 分摊管理

经贝分摊管理页有：

- 分摊数据
- 无分摊数据
- 管理分摊规则
- 字段管理
- 已分摊金额、待分摊金额

我们应把分摊从报表公式里独立出来，形成规则层。

第一阶段建议支持三类规则：

1. 按收入占比分摊。
2. 按人数或人次分摊。
3. 按固定比例分摊。

适用场景：

- 管理中心费用分摊
- 总部公共费用分摊
- 非学科成本费用分摊
- 跨校区项目成本分摊

## 六、数据采集助手借鉴方案

经贝的数据采集助手是机器人流程：

1. 选择机器人。
2. 设置机器人任务。
3. 自动执行。
4. 执行日志。
5. 执行报告。
6. 即时消息通知。

我们暂时不做 RPA，但可以借鉴这个流程，把数据导入升级成向导：

1. 选择数据类型。
2. 上传文件。
3. 字段识别。
4. 公司和期间识别。
5. 导入预检。
6. 入库。
7. 生成导入报告。
8. 进入核算记录。

这个比现在单纯上传文件更适合长期使用。

- 主要字段包括预算项目、产品/项目、计费周期、数量、金额、预算比率。

### 7.2 我们已有公式基础

我们已有 `docs/新项目盈亏平衡测算_样板文件拆解.md`，其中已经记录：

```text
边际贡献率 = 1 - 教师绩效比例 - 平均提成率
盈亏平衡收入 = 基础固定成本 ÷ 边际贡献率
保本学生数 = 盈亏平衡收入 ÷ 平均每生每月收费
```

经贝给我们的启发是：测算模块可以先做成“可保存的测算表”，不必一开始就完全表单化。

### 7.3 第一版测算模块建议

第一版做“新项目盈亏平衡测算”：

1. 方案管理
   - 新增测算
   - 保存测算
   - 复制测算
   - 标记为预算草案

2. 收入假设
   - 产品/项目
   - 计费周期
   - 数量
   - 单价
   - 金额
   - 收入占比

3. 可变成本
   - 教师课时费用
   - 招生提成
   - 市场销售费用
   - 教材物资费用
   - 税金

4. 固定成本
   - 租金物业
   - 基础薪酬和福利
   - 折旧摊销
   - 运营费用

5. 输出指标
   - 边际利润
   - 边际贡献率
   - 经营利润
   - 利润率
   - 盈亏平衡收入

## 八、分阶段开发计划

### 阶段一：界面骨架和筛选标准化

目标：先把系统变成清楚的经营工作台。

任务：

- 完成一级模块导航和二级菜单。
- 统一首页和报表页筛选条。
- 首页筛选支持统计周期、经营口径、经营单元。
- 报表页统一放置导出按钮。
- 保留 AI 客服功能空位，但不开发。

交付结果：

- 经营中心、数据中心、财务中心、基础设置四大模块。
- 首页驾驶舱筛选区更接近经贝样式。
- 用户可以清楚知道每类功能在哪里。

### 阶段二：管理报表成体系

目标：把已有报表升级为经营分析报表。

任务：

- 新增贡献式利润表页面。
- 支持单元视角和月度视角。
- 增加预实对比视角。
- 整理产品线贡献表需求。
- 整理资金变动表口径。

依赖：

- 预算数据。
- 多期间收入和利润数据。
- 公司/板块维度。

### 阶段三：核算记录和分摊管理

目标：让报表结果可以追溯。

任务：

- 建立核算记录只读页面。
- 把导入批次、事实数据、科目映射、业务维度串起来。
- 建立分摊规则表。
- 支持按收入、人数、固定比例分摊。
- 建立已分摊和待分摊列表。

依赖：

- 标准科目表。
- 科目映射表。
- 收入人次或人员数据。
- 管理中心和非学科费用明细。

### 阶段四：数据采集助手

目标：把导入流程从“上传文件”升级为“导入向导”。

任务：

- 选择数据类型。
- 上传文件。
- 自动识别公司、期间、表类型。
- 字段映射和预检。
- 入库和导入报告。
- 失败原因归类。

第一版不做 RPA，只做导入流程体验。

### 阶段五：新项目盈亏平衡测算

目标：把已有 Excel/Word 测算模型系统化。

任务：

- 新增测算方案列表。
- 新增可编辑测算表。
- 保存收入、成本、费用、人员假设。
- 自动计算边际利润、经营利润、盈亏平衡收入和保本学生数。
- 支持方案复制和保存为预算草案。
- 后续支持敏感性分析。

依赖：

- 已有测算样板文件拆解。
- 固定成本和变动成本口径确认。
- 产品线、客单价、人次等业务假设。
