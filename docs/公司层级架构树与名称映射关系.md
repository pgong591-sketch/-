# 公司层级架构树与名称映射关系

## 1. 目标

本模块用于维护集团组织主数据，解决两件事：

- 公司层级架构树：用正式公司编码搭建集团、板块、管理中心、校区/项目的上下级关系。
- 名称映射关系：把源 Excel 里的简称、别名、表头名称统一映射到正式公司编码。

最终口径是：所有财务事实表只存 `company_code`，所有源文件里的公司名称差异先通过 `company_aliases` 归一。

## 2. 核心表

### `companies`

正式公司主数据表。

| 字段 | 用途 |
| --- | --- |
| `code` | 正式公司编码，财务事实表统一引用它 |
| `name` | 公司全称或正式管理名称 |
| `short_name` | 常用简称，导入时也会作为别名 |
| `parent_code` | 上级公司编码，构成父子关系 |
| `level` | 树层级，由系统重建，不手工维护 |
| `tree_path` | 物化路径，例如 `/ROOT/101/10101` |
| `is_leaf` | 是否末级节点，由系统重建 |
| `is_consolidated` | 是否纳入汇总/合并范围 |

### `company_aliases`

源名称到正式编码的映射表。

| 字段 | 用途 |
| --- | --- |
| `alias` | 源文件出现的名称、简称、编码或别名 |
| `company_code` | 映射后的正式公司编码 |
| `source` | 来源，常见为 `companies`、`config`、`manual` |
| `status` | 是否启用，`1` 为启用 |

### `dim_company`

公司业务属性表，用来区分业务模块和经营属性。

| 字段 | 用途 |
| --- | --- |
| `company_id` | 公司编码，对应 `companies.code` |
| `company_name` | 公司名称，跟随 `companies` 同步 |
| `business_group` | 所属业务模块，只允许选择：非学科素质中心模块、尔遇书馆模块、学校模块、幼儿园模块、托育模块、文旅模块、尔遇书城模块、少年宫模块、国际教育模块、职能公司模块、物业公司、外部投资模块 |
| `business_type` | 业态类型，例如小学、初中、高中、校区、书馆、物业、科技公司 |
| `region` | 所属区域 |
| `is_operational` | 是否运营主体 |

### `ownership`

投资关系和直接持股比例表。它不替代组织架构树，而是补充“谁投资谁、占多少股、是否控制”。

| 字段 | 用途 |
| --- | --- |
| `parent_code` | 投资主体或母公司编码 |
| `sub_code` | 被投资主体或子公司编码 |
| `ownership_pct` | 直接持股比例，单位为 `%` |
| `effective_date` | 生效日期，格式 `YYYYMMDD` |
| `expiration_date` | 失效日期，空值表示当前有效 |
| `is_control` | 是否形成控制，`1` 是，`0` 否 |

系统按 `ownership_pct` 和 `is_control` 生成业务分类：

| 规则 | 投资分类 |
| --- | --- |
| `ownership_pct >= 99.999` | 全资子公司 |
| `ownership_pct < 100` 且 `is_control = 1` | 控股公司 |
| `ownership_pct > 0` 且 `is_control = 0` | 参股公司 |
| 无有效比例 | 未配置 |

## 3. 组织分类折叠展示口径

公司层级页面新增“组织分类折叠视图”，展示层分为两大类：

### 实控公司

包括全资子公司、控股公司、实际管理运营公司。系统优先依据 `ownership.is_control = 1` 判断是否控制；没有明确股权记录时，结合 `dim_company.is_operational` 和公司名称/业态作为展示兜底。

实控公司按以下模块折叠展示：

| 模块 | 识别口径 |
| --- | --- |
| 非学科素质中心模块 | `business_group` 已配置为该模块，或公司名称/业态含非学科、素质、虎翼营、拔创、校区 |
| 尔遇书馆模块 | `business_group` 已配置为该模块，或公司名称/业态含尔遇书馆、书馆 |
| 学校模块 | 仅放多维学校相关主体；不能用学校、小学、初中、高中、个性化、培优等泛词自动归类；名称带“校区”的主体默认归入非学科素质中心模块 |
| 幼儿园模块 | `business_group` 已配置为该模块，或公司名称/业态含幼儿园、幼儿 |
| 托育模块 | `business_group` 已配置为该模块，或公司名称/业态含托育 |
| 文旅模块 | `business_group` 已配置为该模块，或公司名称/业态含文旅、探幽 |
| 尔遇书城模块 | `business_group` 已配置为该模块，或公司名称/业态含尔遇书城、书城、书店 |
| 少年宫模块 | `business_group` 已配置为该模块，或公司名称/业态含少年宫 |
| 国际教育模块 | `business_group` 已配置为该模块，或公司名称/业态含国际，或公司编码在国际教育层级下 |
| 职能公司模块 | `business_group` 已配置为该模块，或公司名称/业态含集团、管理中心、科技、职能等职能主体特征 |
| 物业公司 | `business_group` 已配置为该模块，或公司名称/业态含物业 |

未命中的实控公司在折叠视图里临时进入“未分配模块”，便于后续人工确认；“未分配模块”不是可选板块。

### 对外投资公司

包括参与投资但不进行管理运营、只做财务数据跟进的公司或项目。

当前判断口径：

- `ownership.ownership_pct > 0`
- `ownership.is_control = 0`

对外投资公司不再按业务模块拆分，统一按单项目展示；项目行显示投资主体、投资占比、是否控制、投资分类、业态和区域。
对外投资公司可在 `business_group` 选择“外部投资模块”，同时靠 `ownership.is_control = 0` 和 `dim_company.is_operational = 0` 区分。

## 4. 层级树规则

1. 系统使用 `ROOT` 作为逻辑根节点。
2. 公司清单中第一层上级如果是“多维教育集团”等集团名，会映射到 `ROOT`。
3. 每次导入或修改父子关系后，系统重建 `tree_path`、`level`、`is_leaf`。
4. 子树查询通过 `tree_path LIKE '/ROOT/101/%'` 实现，用于母公司自动汇总所有子孙公司。
5. 财务数据仍落在末级或实际上传公司，汇总报表再按树结构向上取数。

## 5. Excel 公司清单导入规则

当前导入函数为 `src.company_hierarchy.import_companies_from_excel`。

支持的核心列：

| 标准含义 | 可识别列名 |
| --- | --- |
| 公司编码 | `公司编码`、`公司代码`、`编码`、`code`、`company_code` |
| 公司名称 | `公司名称`、`名称`、`name`、`company_name` |
| 上级公司 | `上级公司`、`上级编码`、`母公司`、`parent_code`、`parent_name` |
| 简称/别名 | `简称`、`公司简称`、`别名`、`short_name`、`alias` |
| 是否合并 | `是否合并`、`合并范围`、`is_consolidated` |

导入动作：

1. 自动识别表头行，过滤页脚、合计和空行。
2. 用本次文件中的公司名称、简称、编码建立临时映射。
3. 先补齐 `ROOT` 根节点，再写入或更新 `companies`。
4. 导入完成后重建公司树。
5. 从 `companies` 自动种子化别名：公司编码、公司名称、公司简称都会写入 `company_aliases`。

## 6. 名称映射优先级

导入财务报表或预算数据时，解析出的公司名称按以下顺序归一：

1. `company_aliases.alias` 精确匹配。
2. `companies.code`、`companies.name`、`companies.short_name` 精确匹配。
3. 公司全称和简称的包含式模糊匹配。
4. 仍无法识别时返回未匹配，需要补充别名。

别名应保持一对一：同一个 `alias` 不能同时指向多个公司编码。

## 8. 维护建议

- 新公司或组织调整优先维护公司清单 Excel，再重新导入。
- 新的报表简称不要改代码，直接补充到 `company_aliases`。
- 父子关系变更后必须重建树路径。
- 导入前如出现公司识别失败，先检查是否缺别名，再检查是否缺正式公司。
- 业务模块调整维护 `dim_company`，不要改 `companies.parent_code`。
- 投资比例调整维护 `ownership`，默认从当前直接上下级关系生成 100% 基线；真实的占股公司需要财务确认后改为实际比例。
- 对外投资项目需要把 `dim_company.business_group` 维护为“外部投资模块”，把 `ownership.is_control` 维护为 `0`，并把 `dim_company.is_operational` 维护为 `0`，保证展示和后续报表口径一致。
